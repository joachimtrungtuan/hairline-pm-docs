### 13. Screen Specifications Completely Restructured (3-Tenant Format)
- Removed: "Page Header" special section, "Tab Layout" table, and design pattern commentary block
- Replaced with proper 3-tenant screen structure:

  **Patient Platform (2 screens)**
  - Screen 1: In Progress Case View — read-only treatment status, beginning note, day rows with status badges, journey timeline
  - Screen 2: Day Details Popup — single day description and status (read-only; clinical notes hidden from patient)

  **Provider Platform (4 screens)**
  - Screen 1: In Progress Listing — filterable/searchable list of all In Progress cases for the clinic
  - Screen 2: In Progress Case Detail — tabbed case detail (Treatment Plan | Note | Book hotel & Book flight | In Progress); In Progress tab has 3 stages: Before Procedure (beginning note, action buttons), During Procedure (day-by-day rows → Day Edit modal), End of Procedure (3D scan upload, graft count reference)
  - Screen 3: Day Edit Modal — status select + clinical note (provider-only)
  - Screen 4: End of Treatment Modal — conclusion, prescription, advice, medication, actual graft count, final 3D scan, aftercare template

  **Admin Platform (2 screens)**
  - Screen 1: In Progress Listing (Admin) — platform-wide view across all providers with search and filter
  - Screen 2: In Progress Case Detail (Admin) — all fields editable; includes audit trail, manual status transitions, End Treatment and Cancel/Close Case controls, clinician reassignment (logged)
