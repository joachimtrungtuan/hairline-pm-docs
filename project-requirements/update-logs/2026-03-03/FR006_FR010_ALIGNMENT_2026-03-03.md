## Summary

Aligned booking lifecycle statuses and treatment execution rules per product decisions:

- Added `Aftercare` to FR-006 booking status enum and admin/provider status filters
- Clarified FR-010 payment responsibility: payment capture handled in FR-007; FR-010 blocks Check In when outstanding balance exists
- Kept No-Show as an admin-managed case label (not a booking pipeline status)
- Kept medication capture as a single free-text field (no structured schedule generation in FR-010)

---

## Changes Applied

### 1) FR-006 — Booking status enum includes Aftercare

- Updated booking status displays/filters to include: Accepted → Confirmed → In Progress → Aftercare → Completed (plus Cancelled)
- Updated booking status enum definition to include `Aftercare` = treatment complete; aftercare active via FR-011
- Clarified provider "Check In" requires Payment Status = Full paid (no outstanding balance) for handoff to FR-010
- Updated change log to v1.7 and Last Updated to 2026-03-03

### 2) FR-010 — Payment handling aligned to FR-007 + transcriptions

- Removed “final payment triggered on treatment completion” behavior from workflows, dependencies, and requirements
- Added gating: Check In requires no outstanding balance (Payment Status = Full paid)
- Removed direct S-02 payment service integration notes; FR-010 now reads payment state from P-03 / FR-007

**Transcription evidence**:

- `local-docs/project-requirements/transcriptions/Hairline-AdminPlatformPart2.txt` describes split pay with a “full payment needs to be completed before 30 days” cutoff concept (supports payment being completed pre-arrival / pre-treatment start)

### 3) FR-010 — No-Show remains an admin-managed label

- Updated User Story 3 to treat No-Show as a case label/flag with admin reconciliation (no booking status transition to "No-Show")

### 4) FR-010 — Medication remains free-text (Option 2)

- Updated User Story 4 and entity model to reflect medication instructions as a single text field (no “Add Medication” repeater expectations)
- Replaced “medication schedule” wording with “medication instructions” where applicable
